# 🔧 云服务器安全组配置指南

## 问题确认

✅ Ping成功：网络连接正常
❌ 端口无法访问：云服务器安全组未开放端口

---

## 🚨 需要配置的内容

需要在云服务商控制台添加**入站规则**：

### 规则配置

| 配置项 | 值 |
|--------|-----|
| 协议类型 | TCP |
| 端口 | 3000, 9091 |
| 授权对象 | 0.0.0.0/0（允许所有IP访问） |
| 描述 | 记账应用测试端口 |

---

## 📋 配置步骤

### 阿里云

1. 登录 [阿里云控制台](https://ecs.console.aliyun.com/)
2. 找到你的ECS实例
3. 点击"安全组" → "配置规则"
4. 点击"手动添加"
5. 添加规则：
   - 协议类型：自定义TCP
   - 端口范围：3000/3000
   - 授权对象：0.0.0.0/0
6. 再添加一条：
   - 协议类型：自定义TCP
   - 端口范围：9091/9091
   - 授权对象：0.0.0.0/0

### 腾讯云

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/cvm)
2. 找到你的云服务器实例
3. 点击"安全组" → "修改规则"
4. 点击"添加规则"
5. 添加入站规则：
   - 类型：自定义
   - 来源：0.0.0.0/0
   - 协议端口：TCP:3000
6. 再添加一条：
   - 协议端口：TCP:9091

### AWS（亚马逊云）

1. 登录 [AWS控制台](https://console.aws.amazon.com/ec2/)
2. 找到你的EC2实例
3. 点击"Security Groups" → "Edit inbound rules"
4. 点击"Add rule"
5. 添加规则：
   - Type: Custom TCP
   - Port range: 3000
   - Source: 0.0.0.0/0
6. 再添加一条：
   - Port range: 9091
   - Source: 0.0.0.0/0
