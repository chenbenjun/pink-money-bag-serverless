## ❓ 如果还是打不开

### 可能原因1：云服务器安全组未开放端口

需要在云服务商控制台添加入站规则：
- 开放端口：3000、9091
- 协议：TCP
- 来源：0.0.0.0/0（允许所有IP访问）

### 可能原因2：防火墙限制

在服务器上检查防火墙：
```bash
sudo iptables -L INPUT -n
```

如果有防火墙规则阻止了3000和9091端口，需要添加规则。

### 可能原因3：代理软件未完全退出

确保代理软件完全退出，包括系统代理设置：
- Windows: 设置 > 网络和Internet > 代理 > 关闭
- 检查系统托盘，确保代理程序已完全关闭
